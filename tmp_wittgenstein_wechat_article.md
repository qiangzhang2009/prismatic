## 开篇：最难蒸馏的思想家

如果要我选世界上对"蒸馏"最不友好的人，我会选维特根斯坦。

不是因为他思想深刻——思想深刻反而好蒸馏，边界清晰、证据充分。

维特根斯坦难，是因为他**一辈子都在推翻自己**。

早期写《逻辑哲学论》，结论是"凡是可以说的，都可以说清楚；凡是不能说的，必须保持沉默"。

然后他花后半辈子推翻这句话——用日常生活里最普通的例子：尺子用法、棋局、疼痛感觉，说一种叫"语言游戏"的东西。不是一套理论，是一种全新的哲学态度。

从严格逻辑到日常描述，从德语手稿到英语口译文本，前后几乎是两个不同的人。

要把这个蒸馏出来，你面临的挑战不是"理解维特根斯坦"，而是：**如何让代码理解一个不断推翻自己的人类？**

这才是真正的工程问题。

---

## 但让我先告诉你，蒸馏是什么

不做术语解释。直接说我们实际在做什么。

**蒸馏的本质，是从一个人的全部文本中，提取出他的思维骨架。**

不是让人读完全部维特根斯坦再写 prompt。是从原文直接提取：核心概念是什么，他做判断的内在标准是什么，他说话是什么味道，他处理矛盾的逻辑是什么——全部由算法完成，不需要二手理解。

最终输出是一个结构化的 JSON，可以直接注入 AI 对话系统。

你问这个 AI 一个问题，它会用维特根斯坦的方式思考，而不是复述维特根斯坦说过什么。

## 好，现在说那些差点让我们放弃的 bug

蒸馏维特根斯坦这件事，我们真正花时间的部分，不是理解哲学，而是调试代码。

### Bug 1：LLM 永远拒绝说"tensions"

第五步，提取"紧张关系"（认知张力）。

这是理解维特根斯坦最关键的一步——他有一个核心主张：哲学问题应该被"消解"而不是被解答。但同时，他自己在书里做了大量深刻的哲学论证，远不止"消解"。

我们需要一个结构化的 JSON 字段叫 `tensions`，格式是这样的：

```json
{
  "tensions": [
    {
      "type": "belief",
      "claim": "哲学问题应该被消解",
      "counterClaim": "他自己仍在做大量哲学论证"
    }
  ]
}
```

LLM 的回复永远是：

```json
{
  "internalTensions": [...]
}
```

我们加了提示词强调"必须是 tensions"，它改成了 `tensions`。我们加了 example 输出，它跟着 example 学。我们加了"❌ internalTensions 是错误的"，它还是 `internalTensions`。

最后我们放弃说服它，在解析层加了 fallback：无论返回 `internalTensions`、`items` 还是裸数组，全部归一化成 `tensions`。

**教训：LLM 有自己的格式偏好，你改变不了它，就接受它。**

### Bug 2：蒸馏出来的维特根斯坦，脑子里有《圣经》

语料库里有 23,000 多个文件。维特根斯坦的部分，有一页德语手稿扫描质量极差——OCR 识别成了希腊文。然后这页希腊文和另一份希腊文《圣经》的内容混在一起了。

蒸馏结果是：维特根斯坦的"知识图谱"里，莫名其妙多了一大段《圣经》原文引用。

我们后来给它起了个名字：**样本污染**（Sample Contamination）。

解决方法是质量加权采样（Quality Weighted Sampling）：

每个文件加载前先评估质量分数。低质量的自动降权，低于 0.3 分的直接丢弃。同时在文件名层面做过滤——跳过包含"小说""fiction""romance"的文档，防止任何无关内容混入。

从那以后，我们有了第一条语料清洗铁律：**宁可少用，不要错用。**

### Bug 3：济群法师被误诊为"临床"

第四步，语料类型自动检测。

济群法师是一位佛教僧侣。但语料里有很多治疗性的内容——"如何对治焦虑""如何处理情绪"——这些词会触发"临床"类型的检测阈值。

结果：蒸馏引擎把济群法师归类为"临床"类型，用了给医生设计的提取提示词。出来的结果充满了医疗建议，完全不像一个佛教导师。

解决方法是放弃基于关键词频率的简单判断，改用 LLM 本身做类型推断——喂给 LLM 5000 字符的语料样本，让它判断类型，并输出置信度。只有当置信度 >= 0.6 时，才采纳 LLM 的判断。

**教训：最聪明的判断，往往来自另一个 AI，而不是规则引擎。**

### Bug 4：表达层精炼永远不触发

第一步加载语料后，我们会评估还有多少 LLM 预算剩下，用于触发"表达层精炼"——这一步负责把粗糙的词汇提取精修成真正的"说话味道"。

原始代码里，预留预算是这样判断的：

```javascript
if (session.remainingBudget > 1) {
  await refineExpressionWithLLM();
}
```

问题是：知识提取消耗约 $0.017，剩余预算不足 $1。表达层精炼永远不触发。

我们花了三天才发现这个问题。因为日志里没有任何报错——它"成功"完成了，但表达层是空的。

修复后，预算判断改成 `remainingBudget > 0.5`，并加了显式的步骤日志。日志里从此多了一条：

```
[Step 6] Expression refinement triggered. Budget remaining: $0.XX
```

**教训：在管道里，每个步骤都必须有可见的输出。没有输出，就等于没有执行。**

### Bug 5：中国古典哲学差点让服务器 OOM

蒸馏 chinese-classics 语料库（459 个文件，311MB）时，服务直接崩溃了。

错误信息：`FATAL ERROR: invalid array length`。

原因是：正则表达式匹配时，数组长度超过了 V8 引擎的限制。对于超过 500 个文件或总大小超过 10MB 的语料，必须使用分层采样（Stratified Sampling）。

我们后来引入了硬性上限：最多加载 500 个文件，总字符数上限 10M，超出部分按质量分数分层抽样。

**教训：工程上的 OOM，往往是产品定义的失败——如果你不定义上限，大自然会替你定义。**

## 那为什么还要做维特根斯坦

因为有些人的价值，不在于蒸馏评分的数字，而在于他代表的那种思维方式。

维特根斯坦的核心思维习惯，是"先看，不要想"。

遇到一个模糊的概念——"自由""正义""幸福"——大多数人的第一反应是：去找定义。但维特根斯坦会问：这个词在这个语境里是什么意思？说话的人用这个词在做什么？不同场景下用同一个词，是同一个意思吗？

这个思维习惯，可以用在任何领域。

你是一个产品经理，你说"用户体验不好"——先看，你说的"体验"具体指什么？是打开速度？操作流程？还是某种模糊的不适感？

你是一个投资人，你说"这个赛道没有壁垒"——先看，你说的"壁垒"是什么？是技术专利？是品牌？是网络效应？还是你只是在用一个词来掩盖你没有真正分析过这个市场？

蒸馏的意义，就是把这些藏在文本深处的思维方式，提取出来，变成可操作的认知工具。
